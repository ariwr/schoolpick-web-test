# ⚡ 빠른 해결 방법 (5초 안에 해결!)

## 방법 1: PowerShell 환경변수로 즉시 설정 (가장 빠름) ⭐

**비밀번호를 알고 있다면:**

현재 실행 중인 PowerShell 창에서 다음 명령어 실행:

```powershell
$env:DATABASE_PASSWORD="실제비밀번호"
$env:DATABASE_URL="postgresql://postgres:실제비밀번호@3.35.3.225:5432/hw_project001"
```

그 다음 **서버를 재시작**하면 바로 작동합니다!

---

## 방법 2: 비밀번호 찾기 (1-2분 소요)

### A. 팀원에게 물어보기 (가장 확실)
- 팀 채팅방: "hw_project001 DB 비밀번호 알려주세요"
- 예상 소요 시간: 1-2분 (답변 대기)

### B. 다른 프로젝트 확인
- 같은 서버(3.35.3.225)를 사용하는 다른 프로젝트 폴더 열기
- 그 프로젝트의 `.env` 파일에서 `DATABASE_PASSWORD` 확인
- 예상 소요 시간: 30초-1분

### C. 일반적인 비밀번호 시도 (테스트)
다음 비밀번호들을 순서대로 시도해볼 수 있습니다:
- `password` (현재 기본값)
- `postgres`
- `123456`
- `admin`
- 빈 문자열 `""`

---

## 방법 3: 자동 테스트 스크립트 (비밀번호 모를 때)

`test_password.py` 스크립트를 실행하여 일반적인 비밀번호들을 자동으로 테스트할 수 있습니다.

---

## ⚠️ 중요한 것

1. **환경변수 설정은 현재 PowerShell 창에서만 유효합니다**
   - 서버를 실행하려면 환경변수를 설정한 **같은 창**에서 `npm run dev` 실행해야 합니다
   
2. **.env 파일 수정은 나중에 해도 됩니다**
   - 환경변수가 우선순위가 높으므로 .env 파일이 없어도 작동합니다
