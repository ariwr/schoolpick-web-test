# 데이터베이스 비밀번호 찾기 가이드

## 현재 상황
`.env` 파일에 `DATABASE_PASSWORD=password`로 설정되어 있습니다.
이것이 실제 비밀번호인지 확인이 필요합니다.

## 확인 방법 (순서대로 시도해보세요)

### 방법 1: 팀원/관리자에게 문의 (가장 확실한 방법)
- 프로젝트 팀장이나 데이터베이스 관리자에게 문의
- "hw_project001 데이터베이스의 postgres 사용자 비밀번호가 뭔가요?" 라고 물어보기

### 방법 2: 다른 프로젝트 파일 확인
- 같은 서버(3.35.3.225)를 사용하는 다른 프로젝트가 있는지 확인
- 그 프로젝트의 `.env` 파일이나 설정 파일 확인
- 같은 데이터베이스라면 같은 비밀번호를 사용할 가능성이 높음

### 방법 3: 서버 관리자에게 문의
- EC2 서버 관리자나 인프라 담당자에게 문의
- 데이터베이스 서버 설정 문서 확인

### 방법 4: 기본 비밀번호 확인 (임시)
- 개발 환경이라면 기본 비밀번호가 `password`일 수도 있음
- 하지만 프로덕션 환경이라면 보안상 다른 비밀번호일 가능성이 높음

## 현재 해야 할 일

1. **`.env` 파일 열기**
   - `backend-teacher/.env` 파일을 메모장이나 코드 에디터로 엽니다

2. **비밀번호 확인**
   ```env
   DATABASE_PASSWORD=password  # 이것이 실제 비밀번호인지 확인
   DATABASE_URL=postgresql://postgres:password@3.35.3.225:5432/hw_project001
   ```

3. **비밀번호가 다르다면 수정**
   ```env
   DATABASE_PASSWORD=실제비밀번호  # 여기를 실제 비밀번호로 변경
   DATABASE_URL=postgresql://postgres:실제비밀번호@3.35.3.225:5432/hw_project001
   ```

4. **서버 재시작**
   - `.env` 파일 수정 후 서버를 재시작해야 변경사항이 적용됩니다

## 빠른 확인 방법

다음 명령어로 연결 테스트를 해볼 수 있습니다:
```powershell
python test_db_connection.py
```

- 연결 성공: 비밀번호가 맞는 것입니다 ✅
- "password authentication failed": 비밀번호가 틀렸습니다 ❌
